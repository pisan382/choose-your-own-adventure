Done! I've completely rewritten the Implementation Plan section in Brainstorm.md to prioritize cot-story-graph.mmd as the canonical source of truth for all links and edges.

Key changes to the architecture:

Phase 1 now introduces mmd_loader.py — a dedicated MMD parser/reader/writer — and treats story_parser.py as a suggestion engine only.
/api/graph returns edges parsed directly from cot-story-graph.mmd, not from text parsing.
Saving text (POST /api/pages/{page_num}) only writes the .txt file — it never auto-updates the graph.
New endpoints like GET /api/pages/{page_num}/suggestions and POST /api/graph/rebuild let authors see parser hints and explicitly confirm before the MMD is ever overwritten.
Reader Mode linkifies choices only if the MMD confirms the edge exists, falling back to sequential continuation otherwise.
Graph UI shows both "MMD Outgoing Edges" and "Parser Suggestions" in the side panel, with individual "Add Edge" buttons for manual approval.
Parsing is now strictly advisory; the MMD file is only mutated by explicit author actions.
