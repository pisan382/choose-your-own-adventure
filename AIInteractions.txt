Fixed! The problem was in reader.js: the choice links were using absolute paths like href="/#3" instead of relative hashes like href="#3".

What was happening
When you're on http://localhost:8000/reader.html#2 and click a link to /#3, the browser navigates away from reader.html to the root path / (with hash #3). The server serves index.html, which redirects to reader.html. Depending on the browser, that redirect may drop the hash, so you end up at plain reader.html with no hash — which defaults back to page 2.

The fix
I changed all in-story choice links to use relative hashes (#3, #4, etc.) so they stay on reader.html and only change the hash:

// Before (wrong)
href="/#3"

// After (correct)
href="#3"
This applies to:

Regex-detected choice links in the text
Extra MMD edge links
Sequential fallback "Continue to next page" links
Why you start on page 2
The story The Cave of Time begins on story page 2 (the first page of actual narrative). Page 1 of the physical book is the title/copyright page. This is documented in Codebase.md:

"The story begins on story page 2 with: 'You've hiked through Snake Canyon once before...'"

So reader.html defaults to page 2 because that's where the adventure starts.
